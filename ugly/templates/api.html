At this point, the API offers three end points: /feeds, /subscribe, and /unsubscribe. These should each do more or less what you'd expect but the specific usage is described below. Each endpoint accepts GET and POST requests and returns a JSON response. You'll get a 403 if you give an invalid token and a 400 if something goes wrong. The base URL for all of the API endpoints is {{ url_for('.index', _external=True) }}. 

GET /feeds
You can use this endpoint to get a list of all of the feeds to which you have subscribed. It doesn't take any argumentsâ€”besides the required token parameter containing your API key.
Sample Response 
{
    "count": 1,
    "feeds": [
        {
            "id": 1,
            "link": "http://xkcd.com/",
            "title": "xkcd.com",
            "url": "http://xkcd.com/atom.xml"
        }
    ]
}
GET /subscribe
POST /subscribe
This endpoint lets you programmatically subscribe to a feed. This has one required parameter url that should be the URL of the feed. The server will attempt to find the associated feed if the provided URL doesn't return a valid feed. In other words, a call to /subscribe?url=xkcd.com will successfully subscribe you to the feed at http://xkcd.com/atom.xml and return the following response.
Sample Response 
{
    "feed": {
        "id": 1,
        "link": "http://xkcd.com/",
        "title": "xkcd.com",
        "url": "http://xkcd.com/atom.xml"
    },
    "message": "Successfully subscribed to xkcd.com."
}
GET /unsubscribe/<id>
POST /unsubscribe/<id>
This endpoint allows you to unsubscribe from a feed. The value <id> in the endpoint URL above should be a valid feed ID in your subscription list (as returned by the /feeds endpoint above).
Sample Response 
{
    "message": "Successfully unsubscribed from xkcd.com."
}
